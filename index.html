Extension Points

IContentProvider

Purpose: The Content Provider is responsible for the loading and transformation of content models from the underlying domain (CMS)

Base:Sdl.Web.Mvc.BaseContentProvider

Scenario: If a customer doesn't want to use DD4T, you could publish content in another format, and create your own Content Provider to load and transform this data into the View Models

Note: DD4T Content Provider currently does not implement GetEntityModel and GetEntityContent (DCPs)

IContentResolver

Purpose: The Content Resolver takes care of any additional resolving required when mapping the domain (CMS) models to the presentation (View) models (mapping Templates to Views, link/RTF resolving)

Scenario: You might want to extend the logic for resolving RTF content (for example embedding linked content), or alter the behaviour of dynamic linking (resolving DCP links to special URLs)

Note: DXA Vnext gets an additional EvaluateEntity method, which can be used for conditional rendering of CPs

IMediaHelper

Purpose: Defines screen breakpoints for responsive design and generation of responsive URLs for media items (images, videos, etc.)

Base:Sdl.Web.Mvc.Html.BaseMediaHelper

Scenario: DXA has two implementations of this for native image resizing and Contextual Image Delivery. You could extend to create media URLs which map to a CDN

IModelBuilder

Purpose: The Content Provider uses a set of Model Builders to map the domain model to the view models

Base:Sdl.Web.Common.Mapping.BaseModelBuilder

Scenario: You could extend the default Builder, or create a custom Model Builder for a very complex model, that required content transformations not handled by the default model builder logic

Custom Controllers

Purpose: You can use CT Metadata to route the processing of Component Presentations to custom controllers other than the default EntityController

Base:Sdl.Web.Site.
Areas.Core.Controllers.EntityController

Scenario: Search, SmartTarget, Broker Queries, Product data...

This extension pattern is documented at: tridiondeveloper.com
